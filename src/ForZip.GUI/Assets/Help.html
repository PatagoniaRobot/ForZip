Guía de Uso de ForZip

ForZip es una herramienta forense de código abierto diseñada para la compresión, extracción y verificación de integridad de datos.

Funciones Principales

  • Comprimir: Empaqueta archivos y carpetas en formato ZIP (Zip64). Permite cifrado AES-256 y cálculo de hashes simultáneo.
  • Extraer: Descomprime archivos ZIP, incluyendo aquellos protegidos con contraseña.
  • Hash Batch: Calcula hashes (MD5, SHA1, SHA256, SHA512) de múltiples archivos para verificación forense.
  • Verificar: Valida la integridad de un informe (vía .sha256) o re-hashea la evidencia de un ZIP contra su manifiesto, con veredicto archivo por archivo y validación de firma digital.

Uso Forense

Cada compresión genera un informe forense (.report.txt) y un manifiesto (.manifest.json) que incluyen:

  • Metadatos del operador y del caso.
  • Información del entorno (OS, Hostname, Usuario).
  • Hashes individuales, ruta de origen y marca temporal de cada archivo.
  • Hash global del contenedor (ZIP).
  • Archivo de integridad externo .sha256 del informe.
  • Firma digital opcional del manifiesto (CMS/PKCS#7) con el certificado del operador.

Línea de Comandos (CLI)

Ejecute forzip --help en la terminal para ver las opciones de automatización.

ForZip User Guide

ForZip is an open-source forensic tool designed for data compression, extraction, and integrity verification.

Main Features

  • Compress: Packs files and folders into ZIP format (Zip64). Supports AES-256 encryption and simultaneous hashing.
  • Extract: Decompresses ZIP files, including password-protected ones.
  • Hash Batch: Calculates hashes (MD5, SHA1, SHA256, SHA512) for multiple files for forensic verification.
  • Verify: Checks a report's integrity (via its .sha256) or re-hashes a ZIP's evidence against its manifest, with a per-file verdict and digital signature validation.

Forensic Usage

Each compression generates a forensic report (.report.txt) and a manifest (.manifest.json) including:

  • Operator and case metadata.
  • Environment info (OS, Hostname, User).
  • Individual hashes, source path, and timestamp for each file.
  • Global hash of the container (ZIP).
  • External integrity file .sha256 for the report.
  • Optional digital signature of the manifest (CMS/PKCS#7) with the operator's certificate.

Command Line Interface (CLI)

Run forzip --help in your terminal to see automation options.
