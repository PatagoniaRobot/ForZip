Descripción

ForZip es una herramienta forense de compresión y verificación diseñada para garantizar la integridad de la evidencia digital durante su empaquetado y transporte.

A diferencia de las herramientas de compresión convencionales, ForZip implementa un flujo de trabajo forense que calcula los hashes criptográficos (MD5, SHA-1, SHA-256, SHA-512) de los archivos originales durante su inclusión en el contenedor ZIP (un solo pase), asegurando la trazabilidad desde el origen.

Soporta cifrado fuerte AES-256, generación de informes forenses y de un manifiesto legible por máquina, firma digital opcional (CMS/PKCS#7) y verificación de evidencia re-hasheando el contenido del ZIP contra su manifiesto.

Guía de uso

  1. Empaquetar (Comprimir):
    • Arrastre archivos o carpetas al área de trabajo o use el botón de selección.
    • Defina el Nivel de compresión (desde Almacenamiento hasta Ultra).
    • Opcionalmente, active el Cifrado AES-256 y genere una contraseña segura.
    • Seleccione los Algoritmos de hash deseados para documentar la evidencia.
    • Presione "Empaquetar". El hasheado y la compresión ocurren en un solo pase, con feedback en tiempo real.
  2. Extraer:
    • Seleccione el archivo ZIP origen y la carpeta de destino.
    • Ingrese la contraseña si el archivo está cifrado.
    • Durante la extracción, se muestra el progreso y se registran los eventos en la bitácora.
  3. Informe Forense: Al finalizar un empaquetado con hashes, el sistema solicita los datos del operador para generar el .report.txt y el .manifest.json (y, opcionalmente, firmar el manifiesto). El informe documenta metadatos, tiempos, hashes, rutas de origen y marcas temporales de cada archivo.
  4. Verificar: Permite cargar un informe (verifica su .sha256) o un manifiesto .manifest.json para re-hashear la evidencia del ZIP, emitiendo un veredicto archivo por archivo y validando la firma digital cuando existe.
  5. Hash Batch: Utilidad para calcular múltiples hashes de archivos de forma masiva sin necesidad de comprimirlos.
Nota: ForZip utiliza Zip64 de forma predeterminada para soportar contenedores de evidencia que superen los 4GB, asegurando compatibilidad con grandes volúmenes de datos forenses.

Description

ForZip is a forensic compression and verification tool designed to ensure the integrity of digital evidence during packing and transportation.

Unlike conventional compression tools, ForZip implements a forensic workflow that computes cryptographic hashes (MD5, SHA-1, SHA-256, SHA-512) of the original files while including them in the ZIP container (a single pass), ensuring traceability from the source.

It supports strong AES-256 encryption, forensic report and machine-readable manifest generation, optional digital signing (CMS/PKCS#7), and evidence verification by re-hashing the ZIP contents against its manifest.

Usage Guide

  1. Pack (Compress):
    • Drag files or folders to the workspace or use the selection button.
    • Set the Compression Level (from Store to Ultra).
    • Optionally, enable AES-256 Encryption and generate a secure password.
    • Select the desired Hash Algorithms to document the evidence.
    • Press "Pack". Hashing and compression happen in a single pass, with real-time feedback.
  2. Extract:
    • Select the source ZIP file and the destination folder.
    • Enter the password if the file is encrypted.
    • During extraction, progress is displayed and events are recorded in the log.
  3. Forensic Report: Upon completing a packing operation with hashes, the system requests operator details to generate the .report.txt and the .manifest.json (and, optionally, sign the manifest). The report documents metadata, timestamps, hashes, source paths, and modification times for every processed file.
  4. Verify: Load a report (verifies its .sha256) or a .manifest.json to re-hash the ZIP evidence, issuing a per-file verdict and validating the digital signature when present.
  5. Hash Batch: Utility to compute multiple hashes for files in bulk without the need for compression.
Note: ForZip uses Zip64 by default to support evidence containers exceeding 4GB, ensuring compatibility with large forensic data volumes.
