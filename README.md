# ForZip — Forensic ZIP Tool  > Open-source, portable, auditable forensic compression utility with cryptographic verification and AES-256 encryption.  **Version:** 1.2.0   **License:** Apache License 2.0   **Author:** Claudio Andino — Patagonia Robot   **Contact:** claudio@patagoniarobot.com  ---  ## What is ForZip?  ForZip is a free, open-source forensic ZIP utility designed for digital forensic examiners, IT investigators, system administrators, and anyone who needs to package files with verifiable integrity. It produces standard ZIP archives with optional AES-256 encryption and generates a complete forensic report (`.txt`) documenting every operation, including hashes (MD5, SHA-1, SHA-256, SHA-512) of every file processed.  The tool is bilingual (English / Spanish — auto-detected from your operating system), entirely portable (no installation, no registry keys, no telemetry), and completely auditable — every line of source code is published under the Apache License 2.0.  ## Why ForZip?  Most ZIP utilities focus on compression speed or convenience. ForZip is built around a different priority: **reproducible, verifiable, court-defensible packaging of digital evidence**. Every operation is logged. Every hash is stored. Every report is accompanied by an external SHA-256 sidecar (`.sha256`) and a machine-readable manifest, so accidental corruption is detected. The manifest can additionally be **digitally signed** (CMS/PKCS#7) with the operator's X.509 certificate (`--sign-cert`), making tampering cryptographically evident. (Note: without a signature, the sidecar detects corruption, not malicious tampering — anyone who edits the report can recompute it.)  ## Key Features  - **Standard ZIP format** — compatible with 7-Zip, WinRAR, Windows Explorer, etc. - **AES-256 encryption** with optional password protection.
- **Digital signature** of the forensic manifest (CMS/PKCS#7) with the operator's X.509 certificate — verifiable independently (e.g. with OpenSSL).
- **Parallel hashing** of file batches for faster processing of many files. - **Cryptographically secure password generator** with configurable length, character classes, and ambiguity exclusion. - **Multi-hash support** — MD5, SHA-1, SHA-256, SHA-512 (any combination, all, or none). - **Forensic report** in `.txt` format — UTF-8 with BOM, includes operator data, full file listing, individual hashes, and a an external SHA-256 sidecar (`.sha256`) plus a machine-readable `.manifest.json` for automated verification. - **Evidence verification** — re-hash the contents of a previously generated archive against its forensic manifest (`.manifest.json`) and get a per-file verdict (OK / altered / missing / extra), plus verification of the archive's global SHA-256. - **Hash batch mode** — hash files without zipping them. - **Six compression levels** — from Store (no compression) to Maximum.
- **Split archives (multi-volume)** — optionally divide the output into fixed-size volumes (`.001`, `.002`, …), 7-Zip style. Each volume is hashed (SHA-256) individually in the forensic manifest, in addition to the global hash of the logical archive; extraction and verification auto-detect and reassemble the volumes transparently. - **Bilingual UI** — Spanish and English, auto-detected from the system locale. - **Modern interface** — dark theme by default, drag-and-drop, real progress by bytes. - **CLI mode** — full command-line interface for automation and scripting. - **Fully portable** — single executable, no installer, no registry, no admin rights. - **Cross-platform** — Windows is the primary target; Linux and macOS supported via Avalonia UI.  ## Disclaimer  ForZip is provided "AS IS", without warranty of any kind, express or implied, including but not limited to the warranties of merchantability, fitness for a particular purpose, and non-infringement.  **ForZip is provided for forensic, educational, and general-purpose use. Download `ForZip.exe` from the Releases page (link will be added once published). 2. Save it anywhere — your desktop, a USB drive, or a forensic toolkit folder. 3. Double-click to run.  That is it. No installer, no admin rights required, no registry keys created.  ## Quick Start  ### GUI Mode  1. Launch `ForZip.exe`. 2. Drag the file or folder you want to package onto the main window. 3. Choose your options (compression level, encryption, hashes, report). 4. Click **Process**. 5. The ZIP and the forensic report are saved next to your input.  ### CLI Mode  ``` ForZip.exe zip -i "C:\Evidence\Case42" -o "C:\Output\Case42.zip" --hash sha256,sha512 --report "C:\Output\Case42.report.txt" --level 5 ```  Run `ForZip.exe --help` for the full list of CLI flags.  ## Building from Source  ForZip is built with .NET 8 and Avalonia UI 11.  ```bash git clone https://github.com/<user>/forzip.git cd forzip dotnet build -c Release dotnet test dotnet publish src/ForZip/ForZip.csproj -c Release -r win-x64 --self-contained true -p:PublishSingleFile=true -p:IncludeNativeLibrariesForSelfExtract=true -o publish/win-x64 ```  The output is a single `ForZip.exe` in the `publish/win-x64/` directory.  ## Documentation  Contributor and pull-request documentation lives in the `Documentation/` directory:  - [`CONTRIBUTING.md`](Documentation/CONTRIBUTING.md) — Bilingual (English / Spanish) contributor guide: project layout, forensic integrity model, development setup, coding standards, and the pull-request checklist.  ## Contributing  Contributions are welcome.
